# 贵金属牛市风控仪表盘 v0

## 目标

本仪表盘不是用来预测每日涨跌，而是服务于两个问题：

1. 贵金属长期牛市判断是否仍然成立。
2. 当前牛市回调期大致处于哪个阶段，还可能持续多久。

核心原则：长期论题、行情时钟、仓位动作分开判断。论题变强不等于加仓；行情变弱也不必然等于长牛破裂。

## 一、三层判断

### 1. 牛市完整性

回答：长期牛市 thesis 有没有破裂？

重点指标：

| 指标 | 正常状态 | 警戒状态 | 破裂/重审条件 |
| --- | --- | --- | --- |
| 10年期实际利率 | 横盘或回落 | 继续上行 | 持续站上 2.5% 且通胀回落、经济不裂 |
| 金价关键区 | 守住 2025 突破区 | 测试 3800-4000 | 跌破 3800 并站稳两周 |
| 央行购金 | WGC季度估算高于150吨 | 估算与已报告数据剪刀差扩大 | 连续两季低于150吨，或残差口径失真扩大 |
| 西方ETF | 流出放缓或转正 | 有序流出 | 流出重新加速至恐慌级别 |
| 软着陆路径 | 未确认 | 油价回落、核心CPI降温、初请低位 | 美联储控通胀成功且经济不衰退 |
| AI生产率分支 | 未确认 | 收入/capex缺口收窄 | 生产率持续高于2.5%，象限四获得续租 |

硬规则：任一“破裂/重审条件”触发，停止用长期牛市叙事解释短期亏损，启动仓位重审。

### 2. 回调阶段

回答：当前是下跌早段、中段、末段，还是牛市已经转坏？

| 阶段 | 市场特征 | 解释 | 操作含义 |
| --- | --- | --- | --- |
| A. 抛物线后杀估值 | 跌破趋势线，实际利率上行，ETF流出 | 回调早中段 | 不把下跌自动视为机会 |
| B. 卖方衰竭 | ETF流出斜率下降，COT洗至低位，利空不再打出新低 | 回调后半段 | 开始准备右侧确认 |
| C. 转折确认 | 实际利率见顶回落，2Y跌回政策利率下方，ETF周度转正 | 新一轮上涨早期 | 增量资金获得许可 |
| D. 软着陆沉寂 | 通胀回落、经济不裂、实际利率维持高位、金价跌破关键区 | 牛市推迟或阶段性失败 | 降低长期仓位假设权重 |

### 3. 仓位风险

回答：重仓状态下，现在最大的风险是什么？

需要分开看三类资产：

| 资产 | 核心风险 | 额外监测 |
| --- | --- | --- |
| 黄金 | 实际利率上行、ETF流出、结构性买盘失效 | DFII10、GLD持仓、WGC购金 |
| 白银 | 黄金方向叠加工商业周期 | 金银比、铜、制造业PMI、实际利率 |
| 金矿股/GDX | 黄金方向叠加股票与信用风险 | GDX/Gold比值、信用利差、纳指、美元流动性 |

金矿股不是黄金替代品。流动性冲击期，它更像高贝塔股票。

## 二、回调持续时间框架

不使用固定日期预测，使用“顶部后时间 + gap压缩信号”。

### 基础时间窗

抛物线后中继型整理的历史样本通常持续约16-26个月。若以2026年1月顶部计，基础转折窗口落在：

- 快速情景：2026年Q3-Q4
- 基准情景：2026年Q4至2027年中
- 拖延情景：2027年下半年以后
- 失败/沉寂情景：跌破关键区后横盘数年

时间窗只能作为背景概率，不能单独触发交易。

### gap压缩信号

回调接近尾声时，至少应看到以下组合：

| 信号 | 含义 |
| --- | --- |
| 初请失业金持续上行至26-28万 | 劳动力市场开始裂 |
| 2年期收益率跌回政策利率下方 | 市场开始定价宽松 |
| 10年期实际利率见顶回落 | 黄金主逆风消失 |
| ETF周度流量转正 | 西方边际买家回归 |
| 鹰派消息/热CPI不再打出新低 | 卖方枯竭 |

若只满足“时间够久”，但这些信号没有出现，回调不能默认结束。

## 三、红黄绿灯规则

### 绿色：牛市完整，回调仍在

条件：

- 金价守住3800-4000区域。
- 10年期实际利率未持续站上2.5%。
- ETF流出没有重新恐慌化。
- 央行购金未连续两季失效。

含义：长期牛市仍可保留，但增量资金等待确认。

### 黄色：回调后半段，准备右侧

条件：

- ETF流出斜率明显下降。
- COT投机仓位洗至低位。
- 金价对利空开始钝化。
- 实际利率上行放缓。

含义：开始提高更新频率，但仍不把“跌得多”当成买点。

### 蓝色：转折确认

条件：

- 第一层领先信号至少两项触发。
- 第二层确认信号至少一项触发。
- ETF周度转正。

含义：增量资金获得纪律许可。

### 红色：牛市重审

任一触发：

- 金价跌破3800并站稳两周。
- 10年期实际利率持续站上2.5%，且核心通胀回落。
- 美联储成功控制通胀，初请仍低位，经济不衰退。
- ETF流出重新加速至恐慌级别。
- 央行购金连续两季低于阈值。
- AI生产率分支确认，象限四获得续租。

含义：停止默认“这是牛市回调”，重新评估仓位风险。

## 四、每次更新的行动指南

### 第0步：先确认时间

每次拉取最新市场数据之前，第一件事是确认“此时此刻”的绝对时间，并写入本次更新记录。至少记录：

- `update_time_local`：本地更新时间，默认使用北京时间/Asia-Shanghai。
- `update_time_utc`：对应UTC时间。
- `data_cutoff`：本次数据实际覆盖到的最后市场时间，例如“美国市场2026-06-19收盘”或“FRED最新至2026-06-17”。
- `market_session_note`：是否遇到美国假期、半日交易、周末、数据发布滞后等特殊情况。

原则：任何一次市场判断都必须绑定一个具体时间戳。没有时间戳的“最新数据”不可复盘，也不可与下一次更新比较。

### 第0.5步：判断轻量更新还是全量更新

除非用户明确要求“轻量更新”或“全量更新”，否则每次更新前先检查各数据源的时间戳与当前时间的差值，再自动决定更新模式。

**数据滞后检查**：

- 对每个指标记录或读取 `last_observation_time` / `source_release_time` / `last_checked_time`。
- 计算 `data_age = update_time_local - last_observation_time`。
- 若数据源本身滞后发布，例如COT、FRED、WGC，应同时记录“数据覆盖到哪一天”，而不是只记录“今天查了”。

**默认判定规则**：

| 更新模式 | 触发条件 | 更新范围 |
| --- | --- | --- |
| 轻量更新 | 用户明确要求轻量；或自动检查后发现只有日频/高敏感数据过期 | 金价、白银、GDX/GLD/SLV、DXY、油价、2Y/10Y收益率、10Y实际利率、Fed funds/SOFR路径、ETF日频持仓（若可得）、重大事件反应 |
| 全量更新 | 用户明确要求全量；或任一周频、半月频、月频、季频、事件型核心指标超过刷新阈值；或发生FOMC、CPI/PCE、非农、地缘/政治尾部、AI/信用压力等重大事件 | 轻量更新全部内容 + 一阶主轴全链条 + K型前哨 + COT/ETF资金流 + WGC/央行购金 + AI/private credit + 政治尾部升格条件 |

**默认刷新阈值**：

| 数据组 | 默认刷新阈值 | 说明 |
| --- | --- | --- |
| 日频高敏感 | 1个市场日 | 价格、收益率、DXY、油价、实际利率、政策路径 |
| 周频 | 7天 | 初请、COT、ETF周流量、信用利差、事件反应日志复核 |
| 半月频 | 14天 | FOMC会间讲话/纪要漂移、政策路径曲线复核、政治尾部关联关系复核 |
| 月频 | 35天 | 核心PCE、CPI、非农、JOLTS、工资、通胀宽度、零售/PMI |
| 季频/低频事件 | 100天或事件触发 | WGC央行购金、生产率、家庭债务、AI capex、私募信贷压力事件 |

**模式选择原则**：如果慢变量未过期，不为了“完整感”每天重拉；如果慢变量过期，不用轻量更新替代全量更新。轻量更新回答“今天市场有没有变”；全量更新回答“框架判断是否需要重估”。

### 后续步骤

1. 确认时间并写入 `dashboard.md` 顶部。
2. 检查各指标时间戳和数据滞后，决定本次是轻量更新还是全量更新，并把模式写入 `dashboard.md`。
3. 拉取/核验最新市场数据。数据顺序先看一阶主轴链条：通胀 + 就业 → 美联储反应函数 → 实际利率 → 金价；再看K型分布前哨、周期确认信号、转折确认信号和专项观测组。
4. 更新 `data/indicators.csv` 和 `data/event_log.csv`。
5. 重新判定牛市完整性、回调阶段、仓位风险和硬触发器。
6. 在结论中明确哪些数据是当前最新，哪些数据存在滞后。

## 六、下一步实现

建议做成三个文件：

1. `dashboard.md`：最新读数、红黄绿状态、结论摘要。
2. `data/indicators.csv`：每次更新的指标历史。
3. `data/event_log.csv`：记录重要数据公布后金价反应，用来判断卖方是否枯竭。

后续可用 Python 自动拉取 FRED 数据，并保留 ETF、WGC、COT、AI capex 等手工校验项。
